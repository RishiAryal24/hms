# Render Deployment Notes

The deployed system is working with these Render services:

- Frontend: `https://hms-frontend-qmkh.onrender.com`
- Backend API: `https://hms-api-qv4m.onrender.com`
- Tenant schema: `butwalhospital`
- Tenant domain: `hms-api-qv4m.onrender.com`

## Required Environment Variables

Set these on the `hms-api` service:

```env
DEBUG=False
PYTHON_VERSION=3.11.9
ALLOWED_HOSTS=.onrender.com
TENANT_SCHEMA=butwalhospital
TENANT_NAME=Butwal Hospital
TENANT_DOMAIN=hms-api-qv4m.onrender.com
CORS_ALLOWED_ORIGINS=https://hms-frontend-qmkh.onrender.com
CSRF_TRUSTED_ORIGINS=https://hms-frontend-qmkh.onrender.com
CORS_ALLOWED_ORIGIN_REGEXES=https://.*\.onrender\.com
HMS_ADMIN_USERNAME=admin
HMS_ADMIN_EMAIL=admin@example.com
HMS_ADMIN_SUPERUSER=True
```

Render should also provide these automatically:

```env
DATABASE_URL=<from hms-db>
SECRET_KEY=<generated by Render>
```

Set this on the `hms-frontend` service:

```env
VITE_API_BASE_URL=https://hms-api-qv4m.onrender.com/api
```

## Deployment Flow

After pushing changes to GitHub:

1. Redeploy `hms-api` if backend code, Django settings, migrations, tenant setup, or API env vars changed.
2. Redeploy `hms-frontend` if frontend code or `VITE_API_BASE_URL` changed.
3. For frontend env var changes, a redeploy is required because Vite reads env vars at build time.

Use this URL for login:

```text
https://hms-frontend-qmkh.onrender.com/login
```

## Free Render Notes

Free Render web services do not provide Dashboard Shell access. Tenant and admin setup is handled during deploy by:

```bash
python manage.py setup_render_tenant
python manage.py setup_render_admin
```

Those commands run from `build.sh`.

## Admin Password

To create the first admin user, set this on `hms-api` and redeploy:

```env
HMS_ADMIN_PASSWORD=<strong-password>
```

Only use this temporarily when resetting the admin password:

```env
HMS_ADMIN_RESET_PASSWORD=True
```

Remove `HMS_ADMIN_RESET_PASSWORD=True` after the reset deploy succeeds, otherwise every future API deploy will reset the admin password again.

## Common Issues

If the frontend shows a CORS error, confirm the backend env vars exactly match the browser origin:

```env
CORS_ALLOWED_ORIGINS=https://hms-frontend-qmkh.onrender.com
CSRF_TRUSTED_ORIGINS=https://hms-frontend-qmkh.onrender.com
```

If frontend requests go to `https://hms-api.onrender.com`, update and redeploy the frontend:

```env
VITE_API_BASE_URL=https://hms-api-qv4m.onrender.com/api
```

If tenant resolution fails, confirm:

```env
TENANT_DOMAIN=hms-api-qv4m.onrender.com
```
